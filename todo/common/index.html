带SFC的NS实例化失败问题定位

kubernetes环境，查看nslcm容器日志

  1. 检查vnf创建是否成功。vnf创建成功是创建sfc的前提

  2. 检查sfcdriver是否注册，sfcdriver的注册路径为： uui-ui->运维管理->外部系统注册->SFC Driver

    URL地址为 MSB_IP:30280/iui/usecaseui/resource-management/#/esr-sfc-driver

  3. 检查sfcdriver注册url是否为api/fhsfcdriver/v1

  4. 检查sfcdriver注册的VIM ID是否正确

如果上述检查无误，nslcm日志显示sfc中任何一类错误，例如create port chain error等，到第2步
